## HOW TO READ THIS DOCUMENT

- **App Opens / App Closes**: Based on confirmed 2025–2026 cycles; Summer 2027 dates are projected.
- **OPT/CPT**: YES = company has confirmed history of hiring on CPT/OPT without needing H-1B sponsor at internship stage. NO = company explicitly requires permanent work authorization. PARTIAL = depends on team/location. CHECK = verify each cycle.
- **ATS**: The platform the company uses to receive applications (affects resume formatting).
- **Cohort Size**: Total intern class (all roles), not just your target roles unless specified.
- **Compensation**: Hourly USD unless stated otherwise.

## SECTION 15 — NOTIFICATION SYSTEM ARCHITECTURE

### Chosen Approach: Two-Layer Free System

#### Layer 1 — GitHub Actions + Python (Greenhouse & Lever companies)
- **What**: Python script runs on a cron schedule (every 4 hours) via GitHub Actions free tier
- **How**: Hits Greenhouse and Lever public JSON APIs directly — no browser/scraping needed
- **Filters**: Keyword match on title + department (internship, summer 2027, ML engineer, Salesforce developer, etc.)
- **Stores**: New job listings in Supabase (free PostgreSQL, 500MB)
- **Notifies**: Telegram bot message with job title, company, location, and apply link
- **Dashboard**: Streamlit app (Streamlit Community Cloud — free) with interactive filters
- **Cost**: $0

#### Layer 2 — Self-hosted changedetection.io (Workday & JS-heavy companies)
- **What**: Open-source tool (`dgtlmoon/changedetection.io`) self-hosted via Docker
- **Where**: Fly.io free tier (always-on, 256MB RAM sufficient) or Oracle Cloud Always Free VM
- **How**: Playwright browser mode renders JS-heavy Workday pages
- **Notifies**: Same Telegram bot
- **Cost**: $0

|---|---|---|
| Job fetcher (Greenhouse/Lever) | Python script | Free |
